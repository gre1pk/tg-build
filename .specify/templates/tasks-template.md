## Зависимости и порядок выполнения

### Зависимости фаз

- **Setup (1)**: без зависимостей
- **Foundation (2)**: после Setup — **блокирует** все user stories
- **User Stories (3+)**: после Foundation; параллельно или P1 → P2 → P3
- **Polish**: после нужных user stories

### Зависимости user stories

- **US1 (P1)**: после Foundation, без зависимости от других stories
- **US2 (P2)**: после Foundation; может интегрироваться с US1, но тестируется отдельно
- **US3 (P3)**: аналогично

### Внутри каждой user story

- Тесты (если есть) — сначала, должны FAIL
- Модели → сервисы → эндпоинты/UI
- Story завершена → checkpoint

### Параллельные возможности

- Задачи с [P] в Setup и Foundation
- После Foundation — разные stories разным разработчикам
- Модели и тесты с [P] внутри одной story

## Стратегия реализации

### MVP сначала (только User Story 1)

1. Фаза 1: Setup
2. Фаза 2: Foundation
3. Фаза 3: User Story 1
4. **СТОП и проверка** US1
5. Деплой/демо при готовности

### Инкрементальная поставка

1. Setup + Foundation
2. US1 → проверка → деплой (MVP)
3. US2 → проверка → деплой
4. US3 → …

---

## Заметки

- [P] — разные файлы, без зависимостей
- Метка [Story] связывает задачу с user story
- Каждая story должна завершаться и тестироваться отдельно
- Коммит после задачи или логической группы
- Избегать: размытых задач, конфликтов в одном файле, зависимостей, ломающих независимость stories
